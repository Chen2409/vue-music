1、MVC与MVVM (Vue)的区别以及特点（框架开发模式)

Mvc 与Mvvm 都是一种框架开发模式。
M-V-C指的是:将程序强制分为三部分,模型层(Model)、视图层(View)和控制器(Controller)
特点:数据单向流动的
M-V-Vm指的是:将程序强制分为三部分，模型层(Model）、视图层(View）和视图模型层(View model)
特点:数据双向流动的，View 或Model某一个发生变化，另一个随之更新

2、Vue中的生命周期函数有哪些?

Before Create: Vue对象创建之前

Created: Vue 对象创建之后
Before Mount: View 与Model数据绑定成功之前
Mounted: View 与 Model数据绑定成功之后
Before Update: View 或 Model数据更新之前
Updated: View 或 Model数据更新之后
Before Unmount: Vue对象销毁（View 与 Model解绑）之前（ Vue3.0>Unmounted: Vue对象销毁（View 与 Model解绑）之后

4、v-for列表渲染，遍历数组、对象、整数?

遍历数组: v-for="(item, index) in arr" :key="item" item数组中每一个数据index每一个数据数字下标arr 要遍历的源数组
遍历对象: v-for="(value,key,index) in obj" :key="value" value遍历的属性值key遍历的属性名index每个属性数字下标obj要遍历的源对象
遍历整数: v-for="n in num" :key="n" num 要遍历的整数
注意: key? 唯一性索引，遍历出来的标签都是唯一不重复的

6、条件渲染v-if v-show的区别?

v-if 与 v-show 都是根据条件来控制某一个元素的显示与隐藏，
v-if 根据条件判断是否创建这个元素，v-show不管条件是否成立，都创建此元素，通过CSS的display: block/none控制显示与隐藏。

11、什么是动态路由，动态路由传参的几种形式?

动态路由:把某种模式匹配到的所有路由，全都映射到同个组件。(能够接收不同参数的路由，我们称为动态路由)
路由传参几种形式：
1、 props传参 组件接收props：[“xxx”,”xxx”,”xxx”]
值为布尔值true的形式，值为函数的形式
2、 query传参 组件接收参数：$route.query.xxx $route:路由信息对象
可以使用path也可以使用name 传递参数
3、 params传参 组件接收参数$route.params.xxx
只能使用命名路由name传递参数

15、组件之间的数据通信

组件之间的数据通信分为两种：父子组件数据通信 与非父子组件之间的数据通信
1、 父子组件之间的数据通信：
父-->子 通过props传递数据
子—>父：通过$emit()触发自定义事件的形式传递数据

备注：父子组件之间通信也可以使用插槽slot
2、 非父子组件之间的数据通信（两种方法）
第一种：事件总线Bus的形式实现
第二种：状态管理科vuex实现
